Students

GET

Url For API/Students

This will retrieve all Student information. This will retrive the student's first name, last name, and cin.


Url For API/Students/fname/{fname}

This will retrive all students with the first of the value of {fname}.


Url For API/Students/lname/{lname}

Will retrieve all students by the last name from the value {lname}.


Url For API/Students/cin/{cin}

Retrieves students by the cin and since cin is unique it will only retrieve one student.


DELETE

Url For API/Students/delete/{cin}

This will delete the student by the cin.


POST

Url For API/Students/insert/{fname}/{lname}/{cin}/{email}/{password}

This will add the student with the parameters of {fname} for first name, {lname} for last name, and {cin} for cin.

Added email and password, as for first login and admin they are default to true and false respectively.Since they are new users creating accounts they are first time login


Url For API/Admin/{cin}/{first}/{last}

This endpoint will add a new Admin to the database. Only users that have admin priveleges are able to create new admins.


PUT

Url For API/Students/edit/{fname}/{lname}/{cin}

This will update any information for the student since you can't change the cin of the student this will update the first name/last name.


Classes

GET

Url For API/Classes

Retrives all classes. With the information of subject, classCode (ie. MATH 1200, CS 1010, or PHYS 2100), class name, number of units, if it has any prerequisite, if it has any corequisites, and the description of what the class is.


Url For API/Classes/{subject}

Filter classes by Subject.


Url For API/Classes/classCode/{classCode}

Gets the class with that class code and since the class code is unique this will get only one class.


Url For API/Classes/className/{className}

This will retrieve the class/classes with that given {className}


Url For API/Classes/classUnit/{unit}

Get all classes that with that given amount of unit(s). Will mostly be used to help student get classes of a given unit to fufill full student.


PUT

Url For API/Classes/{subject}/{classCode}/{className}/{unit}/{description}

This will update the class information and since class Code is unique this is change anything else from className, number of units, prerequisite, corequisite, or the description of the class.


POST

Url For API/Classes/{subject}/{classCode}/{className}/{unit}/{description}

This will add a class to the class scheduale witht the given information.


DELETE

Url For API/Classes/{classCode}

This will delete the class with the given {classCode}. Since class Code is unique no other information is needed to delete the class.


Paths

GET

Url For API/Paths

Get all the paths that are in the database regardless of student info or what classes they taking or the year they are taking it in.


Url For API/Paths/{cin}

This will retrieve the paths for a specific student with the cin value for {cin}.


Url For API/Paths/Students/{ClassCode}

Get all the students that are taking that class with the specific {classCode}. Will get all regardless of year taken or who took it.


Url For API/Paths/Semester/{semester}/{year}

This will get all the paths for that given Semester (ie. Summer, Spring, Fall, or Winter) and year (year is in 2 digits of the year, ie.2018 will have 18 only).

This will now return in the json semester and year as seperate values

*This endpoint was changed as of February 27, 2018


Url For API/Paths/{cin}/{semester}/{year}

Get the classes for the specific student with value {cin} and the classes they are taking for that specific semester and year.

This will return now in the json semester and year as seperate values

*This endpoint was changed as of February 27, 2018


Url For API/Paths/taken/{cin}

This returns a list of courses that have been taken by the student. These are courses that the student has received a grade for. This list will be used by the algorithm to determine the roadmap.


Url For API/Paths/notTaken/{cin}

This returns a list of courses that the student has not taken or has not received a grade for..


Url For API/Paths/{cin}/{semester}/{year}

This returns the list of courses that the student is taking at a specific semester and year. This is used in the roadmap to show individual semesters.


POST

Url For API/Paths/{cin}/{classCode}/{semester}/{year}

This will create a new path for the student with the class they are taking and what semester they will take it in.


Url For API/Paths/{sid}/{classCode}/{semester}/{year}/{grade}

This will insert a new path for the student if they have taken the class before. Same as the above request only with {grade} as the new entry. This will be used for the new student or students that start the app that have taken any classes before.


DELETE

Url For API/Paths/{cin}/{classCode}/{semester}/{year}

This will delete the path of the student if they do not want to take the class for that semester and year.


PUT

Url For API/Paths/{sid}/{classCode}/{semester}/{year}/{grade}

This one will be for update. So after the semester is over the student will enter a grade they got for that class.

Algorithm

GET

Url For API/Algorithm/{cin}

This calls the Algorithms methods to produce a roadmap. This also uses the classes/taken methods to retrive a list to accurately display a correct roadmap with any classes taken not showing up.


Url For API/Algorithm/{cin}/{semester}/{year}/{classCode}

The frontend uses this when the user edits the roadmap by inserting a class into the roadmap. This endpoint sends the class to the algorithm and makes any changes to make that course into the semester and year they user wanted. same as previous endpoint- it uses the classes/taken method to see the previous classes taken. Returns a roadmap with previous classes taken not showing and the class they edited into the roadmap into the place they wanted.


Url For API/Algorithm/consume/{cin}

This is used for testing purposes to see if the endpoint can properly consume and produce a JSON object. We could not get it working even with the syntax being corrent in the API and in the frontend.


POST

Url For API/Algorithm/{cin}/classinfo

This endpoint is incomplete. This was suppose to consume an incoming JSON object and parse to the appropriate object. In this case it was a roadmap object. The object was going to be used to bulk save the roadmap into the database. This was to reduce time instead of making opening and closing connections for each course.
